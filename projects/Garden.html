Smart Garden Simulation

  • Tech Stack: Java, OOAD, OOP, SOLID, Design Patterns, Layered Architecture, Git, GitHub, IntelliJ IDEA.
  • Github URL: Project Link

Engineered an OOAD-driven Smart Garden Simulation in Java with a centralized GardenManager orchestration layer that executes deterministic simulation “ticks” to update environment state, sensor telemetry, plant growth/health models, and irrigation actions.

Implemented an event-driven interaction model using Observer (pub/sub) for sensor → controller notifications, Strategy for pluggable watering policy selection (e.g., threshold/rule-based decisions), Factory for runtime instantiation of plant/sensor modules, and Singleton for shared configuration/ state to reduce coupling and improve extensibility.

Applied SOLID + layered separation (Controller / Service / Domain / Utility) to isolate business rules from domain entities, enabling rapid addition of new sensor types, irrigation zones, and plant behaviors with minimal refactoring and improved maintainability.
